# OTel SPEC Update — powerloop progress

## Research Notes (from Plan phase)

AI SDK telemetry essentials relevant to SPEC decisions:

- Activation is per-call: pass `experimental_telemetry: { isEnabled: true }` to `generateText` / `streamText` / `ToolLoopAgent.generate()`. It is opt-in and marked experimental in AI SDK.
- Identification: `functionId` (string) and `metadata` (flat record of key/values) surface on spans as `ai.telemetry.functionId` and `ai.telemetry.metadata.*`.
- Privacy/performance toggles: `recordInputs` / `recordOutputs` (both default true) can be turned off to omit prompt text and generated text from spans.
- Custom tracer: pass `tracer` to use a non-global OpenTelemetry `TracerProvider`; otherwise the `@opentelemetry/api` global is used.
- Span taxonomy for `ToolLoopAgent.generate()` (built on `generateText`):
  - `ai.generateText` — full generation; contains 1+ `ai.generateText.doGenerate` and zero+ `ai.toolCall` spans.
  - `ai.generateText.doGenerate` — one provider call.
  - `ai.toolCall` — one tool invocation; attrs include `ai.toolCall.name`, `ai.toolCall.id`, `ai.toolCall.args`, `ai.toolCall.result`.
- Standard attributes: `ai.model.id`, `ai.model.provider`, `ai.usage.promptTokens`, `ai.usage.completionTokens`, `ai.response.finishReason`, plus OpenTelemetry GenAI semantic conventions (`gen_ai.system`, `gen_ai.request.model`, `gen_ai.usage.input_tokens`, `gen_ai.response.finish_reasons`, etc.).
- Integration failures: errors thrown inside telemetry integrations are caught and do not break generation flow (source: telemetry.mdx "Errors inside integrations are caught and do not break the generation flow").
- Shrimp's current `AiSdkMainAgent` (src/infrastructure/ai/ai-sdk-main-agent.ts) already wraps `ToolLoopAgent`; adding OTel is a matter of (a) forwarding `experimental_telemetry` into the agent, and (b) wrapping the `ProcessingCycle` in a parent span so AI SDK's spans nest under it.
- Processing Cycle is the natural parent span: it owns "one heartbeat-triggered unit of work" and the Main Agent call is the only black-box invocation inside it.
- Tracer lifecycle belongs in `infrastructure/` (new `infrastructure/telemetry/`), initialized before the HTTP server starts; exporter configuration is env-driven.
