# Naming Alignment Rename

## Canonical Naming Mapping (authoritative — apply exactly)

Conceptual terms (SPEC.md, docs/architecture.md, CLAUDE.md):

| Old name              | New name                                        |
|-----------------------|-------------------------------------------------|
| Supervisor (= Shrimp) | Supervisor (= internal component, NOT Shrimp)   |
| Task Queue            | Job Queue                                       |
| Processing Cycle      | Job                                             |
| (implicit)            | Job Worker (executes one Job's lifecycle)       |
| Main Agent            | Shrimp Agent                                    |
| Heartbeat ID          | Job ID                                          |
| Fail-Open Recovery    | Fail-Open Recovery (unchanged)                  |
| Shrimp (product)      | Shrimp (unchanged; refers to product/process)   |

Code identifiers (src/, tests/, DI tokens):

| Old identifier                        | New identifier        |
|---------------------------------------|-----------------------|
| MainAgent (port)                      | ShrimpAgent           |
| AiSdkMainAgent                        | AiSdkShrimpAgent      |
| MainAgentInput                        | JobInput              |
| MainAgentInput.heartbeatId            | JobInput.jobId        |
| ProcessingCycle (class/dir/file)      | Job                   |
| use-cases/processing-cycle.ts         | use-cases/job.ts      |
| ports/main-agent.ts                   | ports/shrimp-agent.ts |
| ai-sdk-main-agent.ts                  | ai-sdk-shrimp-agent.ts|
| DI tokens MainAgent / ProcessingCycle | ShrimpAgent / Job     |

Telemetry strings:

| Old value                                       | New value     |
|-------------------------------------------------|---------------|
| ai.telemetry.functionId = "shrimp.main-agent"   | "shrimp.job"  |
| gen_ai.agent.name (current value)               | "shrimp.job"-aligned |

Semantic clarifications (carry into rewritten Glossary):
- Supervisor is now an INTERNAL component of Shrimp (manages heartbeat reception, Job Queue, Job Worker lifecycle). Shrimp itself is NOT the Supervisor.
- Job Worker = the executor that takes one Job from the Job Queue and runs its lifecycle (select task, promote, fetch comments, assemble prompts, dispatch Shrimp Agent, release slot).
- Shrimp Agent = the AI execution engine invoked once per Job (replaces "Main Agent"). It does NOT select tasks or assemble prompts.
- Job ID = UUID v7 generated by the Supervisor at Job start; written to gen_ai.conversation.id on the shrimp.job span.
